# Security Policy

## Repository Access Control

This repository is **public** but **protected**. Here's what that means:

### Access Levels

- **Public Users**: Can view, clone, and fork the repository (READ-ONLY)
- **Contributors**: Can submit pull requests (requires owner approval)
- **Collaborators**: Only those explicitly added by the owner can push changes
- **Owner**: Full control over the repository

### How to Report Security Issues

If you discover a security vulnerability in this project:

1. **Do NOT** open a public issue
2. Contact the repository owner directly through GitHub
3. Provide detailed information about the vulnerability
4. Allow time for the issue to be addressed before public disclosure

### Supported Versions

This project is currently under active development. Security updates will be applied to the main branch.

### Security Best Practices for This Repository

1. **No Direct Pushes**: Changes should come through pull requests
2. **Code Review**: All pull requests require review before merging
3. **No Secrets in Code**: Never commit API keys, passwords, or tokens
4. **Dependency Updates**: Regularly update dependencies to patch vulnerabilities
5. **Access Control**: Collaborator access is reviewed and limited

### For Collaborators

If you have write access to this repository:

- Always create feature branches
- Submit pull requests for review
- Never commit sensitive information
- Keep dependencies up to date
- Follow secure coding practices

### Automated Security

This repository may use:

- Dependabot for dependency updates
- GitHub secret scanning
- Code scanning (where available)

---

**Remember**: Being a public repository does NOT mean anyone can modify it. All changes require owner approval.
