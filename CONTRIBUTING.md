# Contributing to HB_Elise

Thank you for your interest in contributing to this project!

## Important: Repository Permissions

**This repository is public but protected.** This means:

- ✅ You CAN view, clone, and fork the code
- ✅ You CAN submit pull requests with suggested changes
- ❌ You CANNOT push changes directly to this repository
- ❌ Changes will NOT be applied without owner review and approval

## How to Contribute

### 1. Fork the Repository

Click the "Fork" button at the top of this repository to create your own copy.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/HB_Elise.git
cd HB_Elise
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes

- Follow the existing code style
- Test your changes thoroughly
- Keep changes focused and minimal

### 5. Install Dependencies

```bash
npm install
```

### 6. Build the Project

```bash
npm run build
```

This will:
- Build CSS with Tailwind: `npm run build:css`
- Build JavaScript with Babel: `npm run build:js`

### 7. Commit Your Changes

```bash
git add .
git commit -m "Description of your changes"
```

### 8. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 9. Submit a Pull Request

1. Go to the original repository
2. Click "New Pull Request"
3. Select your fork and branch
4. Provide a clear description of your changes
5. Submit the pull request

### 10. Wait for Review

The repository owner will review your pull request. They may:
- Approve and merge it
- Request changes
- Decline it with an explanation

## What Makes a Good Pull Request?

- **Clear description**: Explain what changes you made and why
- **Focused scope**: Address one issue or feature at a time
- **Tested code**: Ensure your changes work as expected
- **Clean commits**: Use meaningful commit messages
- **No secrets**: Never include passwords, API keys, or tokens

## Code of Conduct

- Be respectful and professional
- Provide constructive feedback
- Accept that the repository owner has final say on what gets merged

## Questions?

If you have questions about contributing, please open an issue for discussion.

---

**Remember**: Your pull request is a *suggestion*. The repository owner maintains complete control over what changes are accepted into the codebase.
